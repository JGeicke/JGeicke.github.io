Detailed Description

FirebaseManager is a Singleton that controls the connection between game and database.

All functions are stateless. They can only properly be tested in PlayMode tests.

◆ DebugIsPlayModeTest()

void FirebaseManager.DebugIsPlayModeTest ( )

This is called in PlayMode-Test to set the GameManager reference at a later point in the lifecycle of the FirebaseManager.

The FirebaseManager has to be initialized before the GameManager, but it tries to get the GameManager reference in Start().

Because the GameManager has not been initialized yet, the FirebaseManager throws an Exception.

If set to true the reference to the GameManager is set at a later point in the lifecycle.

It should only be used inside of playmode tests to not disrupt the flow of the game.

◆ DebugSetFirebaseInstance()

void FirebaseManager.DebugSetFirebaseInstance ( FirebaseManager  firebaseManager)

This is called in PlayMode-Test to properly set the FirebaseManager instance.

Normally this happens in Awake(), but Awake() is not called in PlayMode-Tests.

If the instance is not set, a multitude of methods does not work properly.

It should only be used inside of playmode tests to not disrupt the flow of the game.
